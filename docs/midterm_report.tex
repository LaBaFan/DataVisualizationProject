\begin{abstract}
本文档为 FoodETA 项目的中期报告初稿。FoodETA 面向外卖配送效率分析场景，计划基于 Kaggle 公开数据集 Food Delivery Time 构建一个多视图可视分析系统。系统从配送时间、配送距离、天气状况、交通状况、订单时间、骑手属性、车辆类型和城市差异等角度展开分析，帮助用户发现影响外卖配送时效的关键因素、识别高延迟订单模式，并比较不同条件下的配送效率差异。本文档主要说明项目背景、数据说明、分析任务、系统设计方案、当前进展和后续工作计划。
\end{abstract}

\section{项目背景}

外卖配送是即时物流中的典型场景。配送时效不仅受到配送距离影响，也会受到天气、交通密度、订单高峰、车辆类型、多单配送和骑手状态等因素共同影响。对于平台运营、配送调度和用户体验分析而言，仅观察平均配送时间往往不足以解释延迟订单的形成原因。

本项目拟构建 FoodETA：外卖配送时效与延迟因素可视分析系统。系统计划通过多视图联动方式展示配送时间分布、距离与时长关系、时段规律、天气与交通影响、骑手和车辆因素以及城市差异。例如：

\begin{itemize}[leftmargin=2em]
  \item 哪些时段的订单更容易出现配送延迟；
  \item 恶劣天气和高交通密度是否会显著增加平均配送时长；
  \item 是否存在短距离但长时间的异常订单；
  \item 多单配送、骑手评分和车辆类型是否会影响配送表现；
  \item 不同城市或区域之间是否存在配送效率差异。
\end{itemize}

本项目面向的主要用户包括课程项目评审者、外卖平台运营分析人员和对即时物流效率感兴趣的数据分析用户。系统当前定位为课程项目原型，重点展示数据清洗、聚合、可视化编码、多视图联动和分析叙事能力。

\section{分析任务定义}

结合外卖配送时效分析场景，当前项目定义以下核心分析任务。

\subsection*{T1：分析配送时间的整体分布}

基于 \texttt{Time\_taken(min)} 或预处理生成的 \texttt{delivery\_duration\_min}，分析外卖配送时长的分布、均值、中位数、长尾订单和异常延迟订单。该任务用于建立用户对整体配送效率的初步认识。

\subsection*{T2：探索距离与配送时间之间的关系}

基于餐厅与配送地点经纬度计算配送距离，分析 \texttt{distance\_km} 与 \texttt{delivery\_duration\_min} 之间的关系，识别“短距离但长时间”或“长距离但高效率”的异常样本。

\subsection*{T3：分析天气和交通状况对配送时效的影响}

比较不同 \texttt{Weatherconditions} 和 \texttt{Road\_traffic\_density} 下的平均配送时长、延迟率和长尾分布，发现恶劣天气或高交通密度下的配送压力。

\subsection*{T4：比较不同时段的配送效率差异}

基于订单时间和取餐时间，分析早晨、午高峰、下午、晚高峰、夜间等时段的订单量、平均配送时长和延迟率。

\subsection*{T5：分析骑手属性与配送表现之间的关系}

基于 \texttt{Delivery\_person\_Age}、\texttt{Delivery\_person\_Ratings}、\texttt{multiple\_deliveries}、\texttt{Type\_of\_vehicle} 等字段，比较不同骑手群体或配送方式下的时效差异。

\subsection*{T6：识别高延迟风险场景组合}

通过组合筛选，例如“晚高峰 + 高交通密度 + 雨天 + 多单配送”或“远距离 + 低评分骑手 + 摩托车/自行车”，发现配送延迟风险较高的典型场景。

\section{数据说明}

\subsection{数据来源}

本项目计划使用 Kaggle 公开数据集 Food Delivery Time：

\begin{itemize}[leftmargin=2em]
  \item 数据集名称：Food Delivery Time；
  \item 数据集链接：\url{https://www.kaggle.com/datasets/rajatkumar30/food-delivery-time}；
  \item 数据来源：Kaggle 公开数据集。
\end{itemize}

该数据集目标是预测配送员从餐厅到顾客位置完成配送所需时间，字段包括配送员年龄、评分、餐厅与配送位置经纬度、天气、交通密度、订单类型、车辆类型、配送时长等。数据具有时间属性、空间属性、订单属性、环境属性和骑手属性，适合用于外卖配送时效与延迟因素分析。

背景材料可参考论文 Food Delivery Time Prediction in Indian Cities Using Machine Learning Models：\url{https://arxiv.org/html/2503.15177v1}。该论文提到其使用的公开 Kaggle 外卖配送数据约包含 45,000 条印度城市外卖配送记录，每条记录包含配送时间相关特征和目标变量 \texttt{Time\_taken(min)}。本项目最终记录数量以实际下载的数据文件为准。

\subsection{数据组织方式}

项目数据计划存放在 \texttt{data/} 目录下：

\begin{itemize}[leftmargin=2em]
  \item \texttt{data/raw/}：存放原始 Kaggle 外卖配送 CSV 文件；
  \item \texttt{data/processed/}：存放清洗、派生字段计算和聚合后的前端可读取数据；
  \item \texttt{scripts/preprocess.py}：计划用于字段统一、缺失值处理、距离计算、延迟标签生成和数据导出。
\end{itemize}

如果完整数据量或字段不适合前端直接加载，项目会生成聚合后的 JSON/CSV 文件用于可视化。

\section{设计方案}

\subsection{总体设计}

FoodETA 计划采用多视图协调的可视分析界面。系统通过总览面板、配送时长分布、距离-时间散点图、时间模式视图、天气交通影响视图、骑手车辆视图、城市/空间视图和详情面板共同支持用户完成延迟发现、因素比较和场景解释。

当前计划使用的技术栈如下：

\begin{itemize}[leftmargin=2em]
  \item 前端框架：React + TypeScript + Vite；
  \item 可视化库：ECharts / D3.js；
  \item 地图可视化：Leaflet / Mapbox GL JS，可选；如果地图实现成本较高，初期先实现城市/区域对比视图；
  \item 数据处理：Python + pandas；
  \item 包管理工具：npm；
  \item 可选工具：scikit-learn，用于简单聚类、异常检测或延迟风险分组。
\end{itemize}

\subsection{主要视图}

\paragraph{Overview Dashboard}
展示总订单数、平均配送时长、中位数配送时长、延迟订单比例、平均配送距离、主要城市/区域、主要天气和交通状态，并展示订单量最多的时段和配送时长最高的条件组合。

\paragraph{Delivery Time Distribution View}
展示配送时长直方图、箱线图或密度分布，支持按城市、天气、交通状况、车辆类型和订单类型筛选，并突出长尾延迟订单。

\paragraph{Distance-Time Scatter View}
横轴为配送距离 \texttt{distance\_km}，纵轴为配送时长 \texttt{delivery\_duration\_min}，颜色编码交通状况或天气，点大小可编码 \texttt{multiple\_deliveries} 或骑手评分。该视图计划支持 brushing，框选异常订单后联动详情面板和其他视图。

\paragraph{Temporal Pattern View}
展示订单量和平均配送时长随小时变化，支持工作日/周末或不同日期聚合，并可设计“小时 x 交通密度”或“小时 x 天气”的热力图，用于发现午高峰、晚高峰、夜间订单模式。

\paragraph{Weather \& Traffic Impact View}
展示不同天气、交通密度下的配送时长和延迟率，可使用分组柱状图、堆叠柱状图、热力矩阵或箱线图，重点比较不同外部环境对配送效率的影响。

\paragraph{Courier \& Vehicle View}
展示骑手年龄、评分、车辆类型、多单配送等因素与配送时长的关系，可使用箱线图、散点图或分组柱状图，用于分析骑手属性和配送方式是否影响配送表现。

\paragraph{Spatial / City View}
如果数据包含可用经纬度，可以展示餐厅位置、配送位置或订单流向；如果地图实现成本较高，可以先做城市/区域层面的对比视图。该视图计划支持点击城市或区域后联动其他视图。

\paragraph{Detail Panel}
点击订单点、时段、天气、交通状态、城市或骑手群体后，展示订单数、平均配送时长、延迟率、距离分布、典型样本等详细统计，支持 details-on-demand。

\subsection{交互设计}

系统计划支持以下交互：

\begin{itemize}[leftmargin=2em]
  \item 时间筛选：按日期、小时或自定义时段筛选订单；
  \item 城市/区域筛选：按 \texttt{City} 或地图区域筛选订单；
  \item 环境条件筛选：按 \texttt{Weatherconditions}、\texttt{Road\_traffic\_density}、\texttt{Festival} 等字段筛选；
  \item 配送方式筛选：按 \texttt{Type\_of\_vehicle}、\texttt{Type\_of\_order}、\texttt{multiple\_deliveries} 筛选；
  \item brushing \& linking：在距离-时间散点图中框选异常订单后，其他视图同步更新；
  \item details-on-demand：点击订单点、天气类别、交通状态、时段或城市后，右侧详情面板展示对应统计；
  \item 多条件组合筛选：支持组合查询，例如“晚高峰 + 雨天 + 高交通密度 + 多单配送”；
  \item 延迟风险高亮：对配送时间明显高于同距离/同时段平均水平的订单或条件组合进行提示。
\end{itemize}

\section{Case Study 初步设计}

\subsection*{Case Study 1：晚高峰高交通密度下的配送延迟}

该案例计划展示 dinner\_peak 和 high traffic 条件下的配送时效差异。用户可以筛选晚高峰和高交通密度订单，比较其平均配送时长和延迟率是否明显高于其他时段，并在距离-时间散点图中查看远距离订单是否更容易形成长尾延迟。

\subsection*{Case Study 2：恶劣天气对配送时效的影响}

该案例可用于展示天气条件对配送时效的影响。用户可以比较 sunny、cloudy、rainy、stormy 等天气条件下的配送时长分布，观察恶劣天气是否导致平均配送时长上升或延迟订单比例增加。

\subsection*{Case Study 3：多单配送与骑手评分对配送表现的影响}

该案例计划结合 \texttt{multiple\_deliveries} 和 \texttt{Delivery\_person\_Ratings}，分析多单配送场景下，骑手评分较高的群体是否保持更稳定的配送时长。

\section{阶段性风险与应对}

\begin{itemize}[leftmargin=2em]
  \item 字段质量风险：Kaggle 数据可能存在字段命名不统一、缺失值、时间格式不一致或异常经纬度。应在 \texttt{scripts/preprocess.py} 中统一字段命名、记录清洗规则，并输出可复现的处理结果。
  \item 地图实现风险：若经纬度质量不足或地图实现成本较高，初期优先实现城市/区域对比视图，保留地图点图或流向图作为扩展功能。
  \item 交互实现时间风险：优先保证配送时长分布、距离-时间散点图、天气交通视图、详情面板和筛选联动等核心交互，再逐步完善视觉样式和扩展功能。
  \item 性能风险：若前端直接加载完整明细数据造成性能问题，将使用预聚合 JSON/CSV 文件，并对散点图点位进行采样或分层抽样。
\end{itemize}
