# CourseLens：高校课程评价与选课行为可视分析系统

## 1. 项目名称

CourseLens：高校课程评价与选课行为可视分析系统

## 2. 项目简介

CourseLens 是一个面向高校课程评价与选课行为分析场景的可视分析系统。系统计划整合课程基本信息、历史选课数据、课程评价文本、难度评分、工作量、推荐度、教师信息和学期变化等多源数据，通过多视图交互帮助用户理解课程体验和教学质量变化。

本项目主要面向以下用户：

- 高校学生：辅助选课决策，发现适合自身偏好的课程。
- 教学管理人员：观察不同学院、课程类别和教师维度下的课程评价特征。
- 任课教师或助教：了解课程评价变化、学生反馈关键词和潜在改进方向。

系统关注的问题包括：哪些课程难度较高但仍被学生推荐、哪些课程工作量较低且评价较好、课程在更换教师或不同学期后评价是否发生明显变化，以及不同学院课程整体负担和推荐度是否存在差异。

## 3. 功能概览

系统当前计划支持以下核心功能：

- 课程总览 Dashboard：展示课程数量、评价数量、平均难度、平均推荐度、学院分布等总体统计信息。
- 课程难度-推荐度散点图：以课程为基本单元，分析难度、工作量、推荐度和综合评分之间的关系。
- 学期变化趋势分析：展示课程在不同学期中的评分、难度、推荐度、选课人数等指标变化。
- 课程评价文本分析：基于评论文本提取关键词、情感倾向和高频反馈主题。
- 课程对比分析：支持选择多门课程，对比其难度、工作量、推荐度、评价数量和关键词。
- 多条件筛选：支持按学院、课程类别、教师、学期、评分区间等条件筛选课程。
- 多视图联动：在不同视图之间同步选中课程、筛选条件和高亮结果。
- 课程详情面板：展示单门课程的基本信息、指标摘要、历史趋势和代表性评价。

## 4. 分析任务

- T1：发现课程难度、工作量、推荐度之间的关系  
  分析课程是否存在“高难度但高推荐”“低工作量但高评分”“高工作量低推荐”等模式，辅助学生理解课程体验差异。

- T2：比较不同学院/课程类型的课程特征  
  对比不同学院、通识课、专业课、实验课等课程类型在平均难度、工作量、推荐度和评价情感上的差异。

- T3：追踪课程在不同学期中的变化趋势  
  观察课程在多个学期中的评分、难度、工作量、推荐度、选课人数和评价情感变化，发现可能与教师调整、课程改革或考核方式变化相关的异常波动。

- T4：结合评价文本辅助课程选择与教学改进分析  
  从评论文本中提取关键词、情感倾向和典型反馈，帮助学生理解课程真实体验，也帮助教学管理者发现课程改进方向。

## 5. 数据说明

本项目计划使用公开课程平台数据、自采问卷数据和模拟历史课程评价数据的组合。由于课程作业阶段可能无法直接获得完整真实数据，系统会保留清晰的数据字段设计，并对模拟数据生成过程进行说明。

计划使用的数据字段包括：

| 字段名 | 说明 |
| --- | --- |
| `course_id` | 课程唯一标识 |
| `course_name` | 课程名称 |
| `teacher` | 任课教师 |
| `department` | 开课院系 |
| `category` | 课程类别，例如通识课、专业课、实验课等 |
| `semester` | 学期 |
| `enrollment` | 选课人数 |
| `avg_score` | 课程平均评分 |
| `difficulty` | 难度评分 |
| `workload` | 工作量评分 |
| `recommendation` | 推荐度 |
| `comment_text` | 学生评价文本 |
| `sentiment` | 评论情感标签 |
| `keywords` | 评论关键词 |
| `created_at` | 评论或记录创建时间 |

数据目录规划如下：

- 当前数据文件放在 `data/` 目录下。
- 原始数据存放在 `data/raw/`。
- 清洗和预处理后的数据存放在 `data/processed/`。
- 数据仅用于课程学习、系统原型开发和可视化分析展示。
- 若后续包含自采问卷数据，将在处理阶段进行匿名化，避免包含可识别个人身份的信息。

## 6. 系统设计

系统当前设计为多视图协调的可视分析界面，主要包括以下视图和交互模块。

### Overview View

Overview View 用于展示课程数据的总体情况，包括课程总数、评价总数、平均评分、平均难度、平均工作量、平均推荐度，以及不同学院和课程类型的分布情况。

### Scatterplot View

Scatterplot View 计划以散点图展示课程难度、工作量、推荐度、平均评分之间的关系。例如，横轴可以表示难度，纵轴表示推荐度，点大小表示选课人数，颜色表示学院或课程类别。

该视图用于帮助用户快速发现异常课程和典型课程群体，例如高难度高推荐课程、低难度低推荐课程等。

### Trend View

Trend View 用于展示单门课程或课程集合在不同学期中的变化趋势。可视化指标包括平均评分、难度、工作量、推荐度、选课人数和情感倾向变化。

### Text Analysis View

Text Analysis View 用于展示课程评价文本的分析结果，包括关键词词云、关键词频率统计、情感分布和代表性评论。该视图计划支持点击关键词后筛选相关评论和课程。

### Course Comparison View

Course Comparison View 支持用户选择多门课程进行并列对比。对比内容包括课程基本信息、评分、难度、工作量、推荐度、选课人数、关键词和评价情感。

### Detail Panel

Detail Panel 用于展示当前选中课程的详细信息，包括课程名称、教师、学院、课程类别、最近学期评价指标、历史趋势摘要和代表性评论。

### 多视图联动设计

系统计划支持以下联动交互：

- 点击散点图中的课程点位后，更新课程详情面板和趋势图。
- 通过 brushing 选择课程集合后，更新课程列表、文本统计和对比候选项。
- 筛选学院、课程类别、教师或学期后，所有视图同步刷新。
- 点击关键词后，筛选包含该关键词的评论和相关课程。
- 在课程对比视图中添加或移除课程后，同步更新选中状态。

## 7. 技术栈

当前计划使用以下技术栈：

- Frontend: React + TypeScript + Vite
- Visualization: ECharts / D3.js
- Data Processing: Python + pandas

具体实现过程中可能根据数据规模、交互复杂度和开发进度对技术选型进行调整。

在项目根目录下执行：

```bash

```

启动成功后，终端会显示本地访问地址，通常为：

```text

```

## 11. 数据预处理

数据预处理脚本计划放在 `scripts/preprocess.py`，示例运行命令如下：

```bash
python scripts/preprocess.py
```

该脚本计划完成以下处理：

- 字段统一：统一课程编号、课程名称、教师、学院、学期等字段名称和格式。
- 缺失值处理：处理缺失评分、缺失评论、缺失课程类别等情况。
- 课程评价关键词提取：从 `comment_text` 中提取高频词和关键词。
- 情感标签生成：为评论生成正向、中性、负向等情感标签。
- 数据聚合：按课程、教师、学院、学期等维度生成统计指标。
- 输出前端可读取的 JSON/CSV 文件到 `data/processed/`。

预处理后的数据将由前端在系统启动时加载，用于各个可视化视图。

## 12. 使用流程

用户使用系统的基本流程如下：

1. 打开系统首页，查看课程数据整体概况。
2. 通过筛选器选择学院、课程类别、教师或学期范围。
3. 在课程难度-推荐度散点图中观察课程分布，发现异常课程或感兴趣的课程群体。
4. 点击某门课程，查看课程详情、历史趋势和代表性评价。
5. 在趋势视图中查看该课程不同学期的评分、难度、推荐度和选课人数变化。
6. 在文本分析视图中查看评论关键词、情感分布和相关评论。
7. 选择多门课程加入对比视图，比较它们的难度、工作量、推荐度和评价内容。
8. 根据分析结果形成选课建议或教学质量分析结论。

## 13. Case Study 初稿

### Case 1：发现高难度但高推荐课程

用户在散点图中选择“难度较高、推荐度也较高”的区域，系统高亮出一组课程。这类课程可能具有较大的学习压力，但学生整体评价仍然较好。

通过查看详情面板和评论关键词，用户发现这些课程的高频词包括“收获大”“体系清晰”“作业多”“老师负责”等。该案例说明，课程难度不一定与负面体验直接对应，一些高难度课程可能因为教学质量高、内容扎实或反馈及时而获得较高推荐度。

### Case 2：发现某课程在不同学期评价变化明显

用户在趋势视图中查看某门课程的历史记录，发现该课程在某一学期后推荐度明显下降，同时工作量评分上升。

进一步查看该学期的评论关键词，发现“作业增加”“考试难”“节奏快”等词频明显上升。若结合教师信息或课程大纲变化，可以进一步分析评价变化是否与教师调整、考核方式改变或课程内容扩充有关。

该案例可用于帮助教学管理人员识别课程体验变化较大的时间点，并进一步定位可能原因。
